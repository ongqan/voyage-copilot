# Voyage Copilot 业务规则规格

## 1. 规则治理原则

- 资格、点数、费用、携伴、儿童、预约、取消和互斥均由规则引擎裁决；
- 规则具有租户、适用计划、版本、生效/失效时间、优先级和审核状态；
- 每次判断保存输入快照、输出、命中规则版本和Trace ID；
- 已发布规则不可原地编辑，只能创建新版本；
- 冲突、过期、缺输入和无规则默认停止交易，不由AI猜测；
- 以下规则示例均为虚拟Demo规则，不代表真实企业权益。

## 2. 规则优先级

从高到低：法律/安全限制 → 租户专属限制 → 会员计划限制 → 权益限制 → 服务点限制 → 推荐策略。相同优先级出现互斥结论时返回 `RULE_CONFLICT`，不得用“最后写入”覆盖。

## 3. 资格规则矩阵

| 规则ID | 条件 | 结论 | 原因码 |
|---|---|---|---|
| ELG-001 | 会员权益有效且余额>0 | 继续评估 | `ENTITLEMENT_ACTIVE` |
| ELG-002 | 权益已过期 | 不可用 | `ENTITLEMENT_EXPIRED` |
| ELG-003 | 剩余次数为0 | 不可用 | `BALANCE_INSUFFICIENT` |
| ELG-004 | 机场/地区不在适用范围 | 不可用 | `LOCATION_NOT_COVERED` |
| ELG-005 | 航站楼不匹配且不可跨区 | 不可用 | `TERMINAL_MISMATCH` |
| ELG-006 | 使用时间不在服务开放时段 | 不可用 | `OUTSIDE_OPEN_HOURS` |
| ELG-007 | 未满足提前预约时限 | 不可用或人工 | `BOOKING_WINDOW_MISSED` |
| ELG-008 | 必要身份信息缺失 | 需补信息 | `IDENTITY_INFO_REQUIRED` |
| ELG-009 | 行程未确认 | 禁止报价 | `TRIP_NOT_CONFIRMED` |
| ELG-010 | 多条有效规则结论冲突 | 人工确认 | `RULE_CONFLICT` |

## 4. 携伴与儿童规则

规则输入统一使用 `adults、children、infants`，儿童年龄按服务所在地规则计算，并保存服务日年龄快照。

| 场景 | 处理 |
|---|---|
| 免费携伴额度内 | 主权益扣减按规则，同行费用为0 |
| 超出免费携伴额度 | 计算额外虚拟费用并逐人拆分 |
| 儿童年龄缺失 | `PARTY_INFO_INCOMPLETE`，禁止最终报价 |
| 服务不接待儿童 | `CHILD_NOT_ALLOWED`，不推荐 |
| 年龄处于边界日 | 以服务所在地日期计算，展示计算依据 |
| 同行人超过服务容量 | `PARTY_SIZE_EXCEEDED`，不推荐或拆分方案 |

## 5. 费用与虚拟点数

1. 先判断资格，再计算点数，再计算额外费用；
2. 金额使用最小货币单位整数存储，展示时格式化；
3. 币种转换不进入MVP，服务费用使用规则指定币种；
4. 报价保存规则、价格、库存和汇总哈希，默认5分钟过期；
5. 确认时重新计算，差异为0才可复用；有差异需用户再次确认；
6. 虚拟点数使用追加台账：预留、扣减、释放、退回，不直接改余额；
7. 同一个订单和动作使用幂等键，重复请求只返回首次结果。

## 6. 推荐硬过滤

候选出现以下任一情况必须过滤：

- `eligible=false` 或资格状态非唯一；
- 行程未确认或所需字段缺失；
- 服务机场/航站楼不匹配；
- 服务关闭、库存为0或库存数据过期；
- 到达服务点、完成服务、前往登机口后缓冲不足；
- 规则未发布、已失效或来源租户不匹配；
- 同行结构不被服务接受；
- 无法确认营业时间或关键位置。

诊断日志保存过滤原因，但用户端只展示有帮助且不泄露内部数据的原因。

## 7. 推荐排序

MVP默认权重：时间适配30、用户偏好20、费用/点数15、履约质量15、距离10、库存稳定性10。总分仅用于同一候选集排序，不向用户展示伪精确分数。

策略要求：

- 同类服务默认最多3项；
- 同分时按时间适配、履约质量、服务ID稳定排序；
- 运营调整权重必须生成策略新版本；
- 曝光记录候选、分数、过滤原因和策略版本；
- 历史偏好缺失时使用中性值，不推断敏感属性。

## 8. 时间线规则

```text
服务可用窗口 = 营业窗口 ∩ 权益窗口 ∩ 库存窗口 ∩ 行程安全窗口
行程安全窗口结束 = 航班出发时间 - 登机缓冲 - 步行时间
服务最早开始 = 预计到达机场 + 值机/托运时间 + 安检时间 + 到店时间
```

默认缓冲为Demo配置，必须按国内/国际、是否托运、儿童、无障碍和航站楼分别版本化。若 `服务最早开始 + 服务时长 > 行程安全窗口结束`，返回 `INSUFFICIENT_TIME`。

## 9. 订单规则

### 状态转换

| 当前状态 | 允许动作 | 目标状态 | 前置条件 |
|---|---|---|---|
| DRAFT | 报价 | QUOTED | 行程确认、资格通过、库存有效 |
| QUOTED | 请求确认 | AWAITING_CONFIRMATION | 报价未过期 |
| AWAITING_CONFIRMATION | 确认 | CONFIRMED | 有效一次性Token、最终校验通过 |
| QUOTED/AWAITING | 过期 | EXPIRED | 当前时间超过报价有效期 |
| CONFIRMED | 请求修改 | CHANGE_PENDING | 新报价和新确认Token |
| CONFIRMED | 取消 | CANCELLED | 取消规则通过、新确认Token |
| CONFIRMED | 完成 | FULFILLED | 模拟履约结果成功 |
| 任意处理中 | 转人工 | MANUAL_REVIEW | 冲突、失败或高风险 |

禁止跳过状态；失败写操作必须保持原状态且不产生不完整台账。

## 10. 确认令牌

令牌绑定：`tenant_id、user_id、order_id、action、quote_hash、expires_at、nonce`。服务端签名、短期有效、只能使用一次。发生报价变化、用户切换、动作变化、令牌过期或已用时，返回明确错误并要求重新确认。

## 11. 转人工与SLA

| 条件 | 等级 | 默认SLA | 自动动作 |
|---|---|---:|---|
| 用户主动要求人工 | 普通 | 4小时 | 创建会话工单 |
| AI连续两次未解决 | 普通 | 4小时 | 附失败意图与答案 |
| 工具连续失败 | 重要 | 1小时 | 停止写操作、附Trace |
| 规则冲突/订单不一致 | 重要 | 1小时 | 锁定相关模拟操作 |
| 身份验证失败 | 重要 | 1小时 | 遮蔽敏感数据 |
| 投诉、赔偿或疑似恶意 | 紧急 | 15分钟 | 指派专门队列 |
| 跨租户或越权告警 | P0安全 | 立即 | 中断会话并通知安全人员 |

## 12. 异常事件规则

异常事件必须包含类型、来源、发生时间、影响对象、严重度、可信度和版本。影响分析使用航班、机场、航站楼、服务时间、服务点和订单状态关联。

处理优先级：确认原订单仍可用 → 同服务调整时间 → 同类服务替换 → 其他服务类型 → 取消并退虚拟点数 → 人工工单。任何用户选择均需新报价和确认令牌。

## 13. 规则发布门禁

- Schema有效；
- 所有黄金用例通过；
- 新规则没有未解释冲突；
- 影响范围、开始/失效时间和回滚版本明确；
- 编辑人与审核人不同；
- 发布、定时生效和回滚都有审计；
- 发布后执行抽样影子评估和错误率监控。
